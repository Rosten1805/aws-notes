> **Importante:** VPC es uno de los servicios más importantes de AWS. Comprenderlo bien es fundamental para trabajar en Cloud y para certificaciones como **AWS Certified Cloud Practitioner**, **Solutions Architect Associate** y **SysOps Administrator**.

# ¿Qué es una VPC?

Una **VPC (Virtual Private Cloud)** es una **red virtual privada** dentro de AWS donde podemos desplegar nuestros recursos (EC2, RDS, Load Balancers, etc.).

Es como tener un **centro de datos privado dentro de AWS**, totalmente aislado del resto de clientes.

Con una VPC podemos controlar completamente la red:

- Elegir el rango de direcciones IP (CIDR).
- Crear subredes.
- Configurar tablas de rutas.
- Añadir gateways.
- Controlar el acceso mediante reglas de seguridad.

## Ventajas

- Aislamiento de recursos.
- Mayor seguridad.
- Control total de la red.
- Organización de la infraestructura.

---

# VPC por defecto

Cuando creamos una cuenta de AWS, automáticamente se crea una **Default VPC**.

Características:

- Tiene acceso a Internet.
- Las instancias EC2 reciben una IP pública automáticamente.
- También reciben:
  - DNS público.
  - DNS privado.

Si no elegimos una VPC al crear una EC2, AWS utilizará la VPC por defecto.

---

# Espacio de direcciones (Address Space)

Cada VPC necesita un rango de direcciones IP.

Ejemplo:

```
10.0.0.0/16
```

Este rango define todas las IP que podrá utilizar nuestra VPC.

### Ejemplo

```
10.0.0.0/16
```

Permite utilizar:

```
10.0.0.0
hasta
10.0.255.255
```

Total:

```
2^(32-16)
=
65.536 direcciones IP
```

---

# ¿Qué significa el /16?

El número después de la barra indica **cuántos bits pertenecen a la red**.

Cuanto:

- Más pequeño sea el número → mayor red.
- Más grande sea el número → red más pequeña.

Ejemplos:

| CIDR | Nº de IPs |
|------|-----------:|
| /16 | 65.536 |
| /20 | 4.096 |
| /24 | 256 |
| /28 | 16 |

En AWS:

- Tamaño mínimo de una VPC: **/28**
- Tamaño máximo de una VPC: **/16**

---

# Subredes (Subnets)

Las subredes permiten dividir una VPC en redes más pequeñas.

Ejemplo:

```
VPC
10.0.0.0/16
```

Podemos crear:

```
10.0.1.0/24
10.0.2.0/24
10.0.3.0/24
```

Cada subred tiene su propio rango de IP.

Las subredes **no pueden solaparse**.

---

# Subred pública

Una **subred pública** es aquella que tiene acceso a Internet.

Normalmente se utilizan para:

- Load Balancers
- Bastion Hosts
- Servidores Web
- NAT Gateway

---

# Subred privada

Una **subred privada** no puede recibir tráfico directamente desde Internet.

Se utiliza para recursos internos como:

- Bases de datos
- Aplicaciones backend
- Microservicios
- Servidores internos

Es una buena práctica alojar aquí la información crítica.

---

# Ejemplo de VPC

```
VPC
10.1.0.0/16

├── Subred pública
│   10.1.0.0/24
│
└── Subred privada
    10.1.1.0/24
```

---

# Direcciones IP reservadas por AWS

AWS reserva **5 direcciones IP** en cada subred.

Por ejemplo:

```
10.0.0.0/24
```

Las IP reservadas son:

| Dirección | Uso |
|-----------|-----|
| 10.0.0.0 | Dirección de red |
| 10.0.0.1 | Router de la VPC |
| 10.0.0.2 | Reservada por AWS |
| 10.0.0.3 | Reservada para uso futuro |
| 10.0.0.255 | Broadcast |

Estas IP **no pueden asignarse** a instancias EC2.

Creamos:

```
VPC
10.1.0.0/16
```

Tiene:

```
65.536 IPs
```

AWS reserva:

```

```

```

```

```

```

```

```


# CIDR

CIDR (**Classless Inter-Domain Routing**) es el sistema utilizado para definir rangos de direcciones IP.

Está formado por:

```
IP Base / Máscara
```

Ejemplo:

```
10.0.0.0/24
```

Donde:

- **10.0.0.0** → IP base.
- **/24** → Máscara de red.

# Arquitectura típica de una VPC

```
Región AWS

┌───────────────────────────────┐
│             VPC               │
│        10.0.0.0/16            │
│                               │
│ AZ 1               AZ 2       │
│                               │
│ Pública A          Pública B  │
│ Privada A          Privada B  │
└───────────────────────────────┘
```

Cada Zona de Disponibilidad (AZ) suele tener:

- Una subred pública.
- Una subred privada.

Así conseguimos:

- Alta disponibilidad.
- Tolerancia a fallos.
- Mejor distribución de recursos.

---

# Conceptos clave para recordar

✅ Una VPC es una red privada virtual dentro de AWS.

✅ Cada VPC tiene un rango CIDR.

✅ Ese rango puede dividirse en subredes.

✅ Las subredes públicas tienen acceso a Internet.

✅ Las subredes privadas no son accesibles desde Internet.

✅ AWS reserva siempre 5 direcciones IP por subred.

✅ Las subredes nunca pueden compartir el mismo rango CIDR.

# Preguntas típicas de examen

### ¿Qué es una VPC?

Una red virtual privada donde desplegar recursos de AWS con control total sobre la red.

---

### ¿Qué diferencia hay entre una subred pública y una privada?

- Pública → puede acceder a Internet.
- Privada → no puede recibir tráfico directamente desde Internet.

---

### ¿Cuántas IP reserva AWS por subred?

**5 direcciones IP.**

---

### ¿Qué significa el /24 de una red?

Que los **24 primeros bits identifican la red** y quedan **8 bits para los hosts**, lo que proporciona **256 direcciones IP** (251 utilizables en AWS).

---

### ¿Puede una VPC tener varias subredes?

Sí.

De hecho, es la arquitectura recomendada.

---

### ¿Pueden dos subredes compartir el mismo rango de IP?

No.

Los rangos CIDR de las subredes deben ser únicos y no solaparse.
