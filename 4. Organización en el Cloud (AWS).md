# Cuentas de AWS

## ¿Qué es una cuenta de AWS?

Una cuenta de AWS es un contenedor que almacena:

- Recursos (EC2, S3, RDS, etc.)
- Identidades (Usuarios IAM, Grupos y Roles)
- Configuración
- Facturación

Cada cuenta dispone de:

- Un usuario Root (raíz)
- Un correo electrónico único
- Un método de pago

> Todo el acceso está denegado por defecto excepto para el usuario Root.

---

# Usuario Root

El usuario Root:

- Tiene control absoluto sobre la cuenta.
- No puede ser restringido mediante IAM.
- Debe utilizarse únicamente para tareas administrativas excepcionales.

Buenas prácticas:

- Activar MFA.
- No utilizarlo para el trabajo diario.
- Crear un usuario IAM administrador.

---

# Usuarios, grupos y roles IAM

Dentro de una cuenta pueden existir:

- Usuarios IAM
- Grupos
- Roles

Cada uno puede tener:

- Permisos completos.
- Permisos limitados.

---

# ¿Por qué utilizar varias cuentas AWS?

AWS recomienda separar entornos para reducir riesgos.

Ejemplo:

```text
Cuenta DEV
Cuenta TEST
Cuenta PROD
```

También pueden separarse por:

- Equipos.
- Productos.
- Clientes.

Beneficios:

- Mayor seguridad.
- Menor impacto ante errores.
- Mejor control de costes.

---

# Single Account

Todos los recursos están dentro de una única cuenta.

## Ventajas

- Administración sencilla.
- Una sola consola.
- Menor complejidad.

## Desventajas

- Un error afecta a toda la infraestructura.
- Mayor superficie de ataque.
- Difícil separar costes.
- Riesgo elevado en producción.

---

# Multi Account

Los recursos se distribuyen entre varias cuentas.

Ejemplo:

```text
Management
Development
Testing
Production
```

## Ventajas

- Aislamiento de recursos.
- Mayor seguridad.
- Permisos independientes.
- Mejor control del gasto.
- Menor impacto ante errores.

## Desventajas

- Administración más compleja.
- Más configuración inicial.

---

# Configuración recomendada para Producción

Para una cuenta de Producción:

- Crear una cuenta independiente.
- Utilizar un correo diferente (ej: Gmail +alias).
- Activar MFA para Root.
- Configurar AWS Budget.
- Permitir acceso a facturación mediante IAM.

---

# AWS Organizations

Servicio global para administrar múltiples cuentas AWS.

Permite:

- Gestionar todas las cuentas desde un único lugar.
- Centralizar la facturación.
- Compartir descuentos por volumen.
- Compartir Reserved Instances.
- Automatizar la creación de cuentas mediante API.

---

# Componentes de AWS Organizations

## Cuenta de gestión (Management Account)

- Controla toda la organización.
- Administra las cuentas miembro.

---

## Cuentas miembro

Son las cuentas que pertenecen a la organización.

Solo pueden pertenecer a una organización a la vez.

---

## Organizational Units (OU)

Permiten agrupar cuentas.

Ejemplo:

```text
Root
│
├── Prod
│
├── Dev
│
├── Marketing
│
└── Ventas
```

Sirven para aplicar políticas comunes.

---

# Service Control Policies (SCP)

Las SCP permiten controlar qué servicios y acciones pueden utilizar las cuentas miembro.

Importante:

- No conceden permisos.
- Establecen el límite máximo de permisos.
- Se aplican a nivel de organización.
- Tienen prioridad sobre IAM.
- No afectan a la cuenta de gestión (Management Account).

---

## Jerarquía de SCP

Las SCP siguen una estructura jerárquica:

```text
Root
│
├── OU
│   ├── Cuenta A
│   └── Cuenta B
```

Las restricciones heredadas afectan a todas las cuentas inferiores.

---

## Ejemplos de SCP

Permitir todo:

```json
Allow *
```

Denegar S3:

```json
Deny s3:*
```

Permitir solo EC2 y S3:

```json
Allow:
- ec2:*
- s3:*
```

---

# Relación entre IAM y SCP

## IAM

Define lo que un usuario **puede hacer**.

## SCP

Define el **máximo permitido** dentro de la organización.

Ejemplo:

```text
IAM permite S3
SCP lo deniega

Resultado:
❌ Acceso denegado
```

La denegación explícita siempre tiene prioridad.

---

# IAM Identity Center (antes AWS SSO)

Servicio recomendado por AWS para autenticación centralizada.

Permite:

- Inicio de sesión único (SSO).
- Acceso a múltiples cuentas AWS.
- Acceso a aplicaciones externas.

Compatible con:

- Active Directory.
- AWS Directory.
- Active Directory local.
- Proveedores SAML 2.0.

---

## Ventajas

- Un único inicio de sesión.
- Gestión centralizada de usuarios.
- Gestión centralizada de permisos.
- Ideal para empresas.

---

# AWS Control Tower

Servicio que automatiza la creación y administración de entornos Multi-Account.

Integra:

- AWS Organizations.
- IAM Identity Center.
- CloudFormation.
- AWS Config.
- Otras herramientas de gobierno.

---

# Componentes principales de Control Tower

## Landing Zone

Configuración inicial del entorno multicuenta.

---

## Guard Rails

Controles automáticos de seguridad y cumplimiento.

---

## Account Factory

Automatiza la creación de nuevas cuentas siguiendo estándares definidos.

---

## Dashboard

Panel central para administrar todas las cuentas.

---

# Buenas prácticas

- No trabajar con el usuario Root.
- Utilizar MFA en todas las cuentas.
- Separar DEV, TEST y PROD.
- Utilizar AWS Organizations para gestionar varias cuentas.
- Organizar cuentas mediante Organizational Units (OU).
- Aplicar SCP para limitar permisos globales.
- Gestionar usuarios mediante IAM Identity Center.
- Automatizar organizaciones grandes con AWS Control Tower.

---
