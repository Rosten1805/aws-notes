# Subnetting desde cero 

## ¿Qué es una dirección IPv4?

Una dirección IPv4 está formada por **32 bits**, divididos en **4 octetos** de **8 bits** cada uno.

Ejemplo:

```text
192.168.1.0

11000000.10101000.00000001.00000000
```

Cada número decimal (192, 168, 1, 0...) representa un octeto de 8 bits.

---

# ¿Qué significa el /24, /26, /27...?

Ese número se llama **prefijo CIDR**.

Indica **cuántos bits pertenecen a la parte de red**.

Ejemplo:

```text
192.168.1.0/24
```

Significa:

* 24 bits son de la red.
* Los 8 bits restantes son para los hosts.

---

# Cómo construir la máscara de subred

## Paso 1

Escribe 32 bits.

Ejemplo (/26):

```text
□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□
```

## Paso 2

Pon tantos **1** como indique el prefijo.

```text
/26

11111111.11111111.11111111.11000000
```

¿Por qué?

* Primer octeto = 8 bits
* Segundo octeto = 8 bits
* Tercer octeto = 8 bits

Ya tenemos:

```text
8 + 8 + 8 = 24 bits
```

Faltan:

```text
26 - 24 = 2 bits
```

Por eso el último octeto queda:

```text
11000000
```

# ¿Cómo pasar de binario a decimal?

Muy fácil.

Solo sumamos los valores donde haya un **1**.

Ejemplo:

```text
11000000
```

Tabla:

```text
Binario:  1   1   0   0   0   0   0   0
Valor:   128 64  32  16  8   4   2   1
```

Sumamos:

```text
128 + 64 = 192
```

Resultado:

```text
11000000 = 192
```

---

Otro ejemplo:

```text
11100000
```

```text
128 + 64 + 32 = 224
```

---

Otro ejemplo:

```text
11111111
```

```text
128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255
```

Por eso:

```text
11111111 = 255
```

---

# ¿Por qué aparece tanto el 255?

Porque:

```text
11111111
```

es el mayor número que puede representarse con un octeto de 8 bits.

Siempre vale:

```text
255
```

Por eso una máscara como:

```text
11111111.11111111.11111111.00000000
```

se convierte en:

```text
255.255.255.0
```

# El truco del salto (Block Size)

Una vez obtenida la máscara, calcula:

```text
256 - último octeto de la máscara
```

Ese resultado es el **salto** entre subredes.

## Ejemplo

Máscara

```text
255.255.255.192
```

Último octeto

```text
192
```

Calculamos

```text
256 - 192 = 64
```

Las subredes empezarán cada 64 direcciones.

```text
0
64
128
192
```

---

Otro ejemplo

Máscara

```text
255.255.255.224
```

```text
256 - 224 = 32
```

Las subredes empiezan en:

```text
0
32
64
96
128
160
192
224
```

# ¿Qué es una dirección de red?

Es la **primera dirección** de una subred.

Ejemplo:

```text
192.168.1.0
```

No pertenece a ningún dispositivo.

Solo identifica la red.

Es como el cartel de un edificio.

---

# ¿Qué es un host?

Un **host** es cualquier dispositivo conectado a la red.

Ejemplos:

* Ordenador
* Servidor
* Portátil
* Móvil
* Impresora
* Cámara IP
* Máquina virtual
* NAS

Ejemplo:

```text
192.168.1.15
```

Puede ser perfectamente un host.

---

# ¿Qué es el broadcast?

Es la **última dirección** de la subred.

Sirve para enviar un mensaje a **todos los hosts** de esa red al mismo tiempo.

No puede asignarse a un dispositivo.

---

# Ejemplo completo

Red:

```text
192.168.1.0/26
```

Máscara:

```text
255.255.255.192
```

Salto:

```text
256 - 192 = 64
```

Subredes:

```text
192.168.1.0
192.168.1.64
192.168.1.128
192.168.1.192
```

Primera subred:

| Concepto    | Dirección    |
| ----------- | ------------ |
| Red         | 192.168.1.0  |
| Primer host | 192.168.1.1  |
| Último host | 192.168.1.62 |
| Broadcast   | 192.168.1.63 |

# Mi método para resolver cualquier ejercicio

1. Leer el prefijo (por ejemplo, `/27`).
2. Escribir tantos **1** como indique el prefijo.
3. Convertir la máscara de binario a decimal usando la tabla `128 64 32 16 8 4 2 1`.
4. Calcular el **salto**: `256 - último octeto de la máscara`.
5. Escribir las subredes sumando ese salto.
6. Identificar:

   * Primera IP = Dirección de red.
   * Última IP = Broadcast.
   * Todo lo que hay entre ambas = Hosts.

## Regla de oro

* **Primera IP = Red**
* **Última IP = Broadcast**
* **Las demás = Hosts**

Si sigo estos pasos siempre, podré resolver ejercicios de subnetting sin memorizar fórmulas complicadas.
